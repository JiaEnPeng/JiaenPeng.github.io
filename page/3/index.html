利用zlib解压http中gzip数据

Posted on 2017-10-19

什么是GZIP

GZIP是一种文件压缩算法，其应用范围广，GZIP压缩的比率往往在3到10倍，也就是本来90k大小的页面，采用压缩后实际传输的内容大小只有28至30K大小，这可以大大节省服务器的网络带宽，同时如果应用程序的响应足够快时，网站的速度瓶颈就转到了网络的传输速度上，因此内容压缩后就可以大大的提升页面的浏览速度。在传统的http协议中，通过采用gzip压缩技术，将传输的数据经过压缩后发送到对端进行解压缩操作，可以大大的减少web服务器与客户端的流量。http的gzip压缩是由WEB服务器和浏览器之间共同遵守的协议，也就是说WEB服务器和浏览器都必须支持该技术，而现在主流的浏览器都是支持的，包括IE、FireFox、谷歌浏览器、Opera 等。常见的WEB服务器有Apache 和IIS 等。

双方的协商过程如下

  1. 首先浏览器请求某个URL 地址，并在请求的头 (head) 中设置属性accept-encoding值为gzip，表明浏览器支持gzip这种压缩方式。

（注：gzip是一种数据压缩格式，默认且目前仅使用deflate算法压缩data部分。）

  1. WEB服务器接收到请求后判断浏览器是否支持压缩，如果支持就传送压缩后的响应内容，否则传送不经过压缩的内容；

  2. 浏览器获取响应内容后，判断内容是否被压缩，如果是则解压缩，然后显示响应页面的内容。（IE5.5以上才支持gzip）

HTTP报文分析

在浏览器收到web服务器的回复时，通过http头部中的Content-Encoding属性，值为gzip，则表明此数据报文是经过gzip压缩的。若http头部中，还包含Transfer-Encoding属性，且其值为trunk时，表明此数据报文是经过gzip压缩后，再封装成trunk格式，一段一段发送过来的，此时，其数据包大小不定，一直到接收到终止trunk为止。

获取HTTP报文的压缩数据

  1. 不带chunk:

当获取到不带chunk的压缩数据报文，此时http除去头部信息后，剩下的字节数都为待解压的gzip数据。

  1. 带chunk:

当获取到带chunk的压缩数据报文，此时http除去头部信息后,剩下的字节都封装成chunk格式，通过解析一段一段的chunk，最终将里面的数据封装成为带解压的gzip数据。

chunk数据格式为：size\r\ndata\r\n，每个chunk都是以size开头，标示该chunk数据域的长度，size与data之间用\r\n隔开，最后再以\r\n标示该chunk的边界。

解压http报文的gzip数据

要解压http报文的gzip数据，必须利用zlib库进行解压，解压方式可分为：

  1. 整体解压：

待http报文全部接收完成，再获取其中的gzip数据，将整段gip数据利用zlib库的inflate函数进行解压。

  1. 分段解压：

不等http报文全部接收完成，将当前获取的gzip数据，利用zlib库的inflate函数进行解压。

注：分段解压时，必须保证先后顺序，且分段间必须连续无间断方可。

Redis数据结构之链表实现——Adlist

Posted on 2017-10-19

介绍

在项目开发过程中，链表成为了不可或缺的一种数据结构，不同的项目有自己不同的实现方式。在Redis中，拥有了一套属于它自己的链表结构，其实现过程在源码的adlist.c和adlist.h文件中。接下来将分析它。

结构说明

由此结构体可知，每个链表节点都包含了一个指向前一个节点的指针和一个指向后一节点的指针，所以，其实现了一个双向链表。在adlist.h文件中，我们还可以看到关于链表的结构体声明，每一条链表，都对应一个该结构体，其原型如下：

由此可知，当使用链表时，为每一个链表开辟一个该结构体空间，用来维护改链表的一系列操作。

链表的创建、添加、查找、删除

众所周知，一个完整的链表实现必须要满足创建、添加、查找和删除等操作，在Redis
中，同样不例外的实现了创建、添加、查找与删除等接口，分别如下：

  1. 创建

    list *listCreate(void);
    通过调用该接口，可以获得一个空链表，此时链表的头节点和尾节点都置为空。

  2. 添加
    list *listAddNodeHead(list *list, void *value);
    list *listAddNodeTail(list *list, void *value);
    list *listInsertNode(list *list, listNode *old_node, void *value, int after);
    Redis的链表支持从头部和尾部两种方式添加节点，当从头部添加节点时，新添加的节点将成为链表的头结点，之前的头结点将成为新节点的下一个节点；当从尾部添加节点时，新添加的节点将成为链表的尾节点，之前的尾节点将成为新节点的上一个节点。同时Redis也支持从特定的节点处添加节点（listInsertNode），当调用listInsertNode函数时，必须确保old_node参数（即想要添加的节点的参照节点）必须存在，否则将出现错误，此接口中，after参数为相对于old_node节点的位置。

  3. 删除
    void listDelNode(list *list, listNode *node);
    通过调用该接口，传入想要删除的节点，即可删除该节点。

  4. 查找
    listNode *listSearchKey(list *list, void *key);
    通过该接口，传入想要查找的关键字，返回匹配该关键字的节点。匹配操作分两种形式，一种为当list的match回调函数成功注册时，将使用用户的注册函数进行匹配，第二种当list的match回调未进行注册时，将用key直接与节点的value进行匹配。

链表的遍历

链表的遍历，是链表必须要有的操作，一套好的遍历接口，可以方便的让使用者不用太关心链表的具体实现细节，即可遍历整个链表。在Redis的链表中，采用了迭代器的方式进行遍历，当用户需要遍历链表时，先申请一个迭代器，然后使用迭代器进行遍历，相当方便。接下来，一起看看Redis是怎样使用迭代器作为链表的遍历手段的。

链表的复制

Redis提供了复制链表的接口，当需要复制一个同样的链表结构时，只需要调用listDup函数即可，其原型如下：

list *listDup(list *orig);

值得注意的是，当连接的dup回调函数为空时，复制的链表，其各节点所有的数据指针都将和原链表的相同，故此时如果对原链表的数据进行操作，将影响到复制的链表，切记。

链表的释放

当需要释放链表时，调用listRelease函数即可，其原型如下：

void listRelease(list *list);

其他相关

在adlist.h文件中，提供了一系列的宏操作，如listLength、listFirst、listLast等，从这些宏定义的名称上就能看出来它们的作用是干嘛的。下面，我们来看看listLength宏的实现：

#define listLength(l) ((l)->len)

通过该宏，用户可以获取到该链表的长度（即节点数），通过分析发现，其实该宏并没做啥特别的操作，直接指向了链表的len属性。可能有人就有疑问了，为啥需要这么做呢，岂不多此一举？答案并非如此，在我们的编码工作中，我们有时候并不希望别人直接访问我们的数据，因而提供一个接口，是调用者能够间接地进行访问而不需要知道我内部的数据结构。使用者可以更加方便的使用，从而更加关注于自己的业务开发。

总结

通过学习Redis链表实现，掌握了链表的一些基本操作以及平常编码过程中的一些技巧。

Mysql按天、周、月、季度、年统计数据

Posted on 2017-10-19

背景

在mysql使用过程中，常常需要将查询的数据按天、周等格式进行统计输出。此时需要用到DATE_FORMAT函数，对结果数据中的日期按特定格式进行分类。

DATE_FORMAT简介

函数原型：DATE_FORMAT(date,format)
参数说明：1. date 日期。 2. format 格式
函数功能：根据format字符串格式化date值。其中format可以是：

  • %M 月名字(January……December)
  • %W 星期名字(Sunday……Saturday)
  • %D 有英语前缀的月份的日期(1st, 2nd, 3rd, 等等。）
  • %Y 年, 数字, 4 位
  • %y 年, 数字, 2 位
  • %a 缩写的星期名字(Sun……Sat)
  • %d 月份中的天数, 数字(00……31)
  • %e 月份中的天数, 数字(0……31)
  • %m 月, 数字(01……12)
  • %c 月, 数字(1……12)
  • %b 缩写的月份名字(Jan……Dec)
  • %j 一年中的天数(001……366)
  • %H 小时(00……23)
  • %k 小时(0……23)
  • %h 小时(01……12)
  • %I 小时(01……12)
  • %l 小时(1……12)
  • %i 分钟, 数字(00……59)
  • %r 时间,12 小时(hh:mm:ss [AP]M)
  • %T 时间,24 小时(hh:mm:ss)
  • %S 秒(00……59)
  • %s 秒(00……59)
  • %p AM或PM
  • %w 一个星期中的天数(0=Sunday ……6=Saturday ）
  • %U 星期(0……52), 这里星期天是星期的第一天
  • %u 星期(0……52), 这里星期一是星期的第一天
  • %% 一个文字“%”
