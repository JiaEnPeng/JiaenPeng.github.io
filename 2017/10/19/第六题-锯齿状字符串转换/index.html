第六题 锯齿状字符串转换

Posted on 2017-10-19

问题

The string “PAYPALISHIRING” is written in a zigzag pattern on a given number of rows like this: (you may want to display this pattern in a fixed font for better legibility)

字符串“PAYPALISHIRING”以给定行数的锯齿形图案写成：（您可能希望以固定字体显示此模式以获得更好的可读性）

P A H N
A P L S I I G
Y I R

And then read line by line: “PAHNAPLSIIGYIR”
然后逐行读取：“PAHNAPLSIIGYIR”

Write the code that will take a string and make this conversion given a number of rows:
编写将采用字符串并使此转换给出多行的代码：

string convert(string text, int nRows);

convert(“PAYPALISHIRING”, 3) should return “PAHNAPLSIIGYIR”.
convert（“PAYPALISHIRING”，3）应该返回“PAHNAPLSIIGYIR”。

分析思路

通过题意，假设我们将字符串分割成N=5行，通过将字符串的索引填入相应的行和列中，那么可以得到如下的一张表：

行\列 第0列 第1列 第2列 第3列 第4列
第0行 0 8 16
第1行 1 7 9 15 17
第2行 2 6 10 14 18
第3行 3 5 11 13 19
第4行 4 12 20

通过分析上表，可以发现一定的规律，可以发现：

  • 处于中间行的元素，其索引值满足以下规律：

    • 1.1 当列为偶数时，其索引值可以用公式计算得到：(N-1)*column + row

    • 1.2 当列为奇数时，其索引值可以用公式计算得到：(N-1)*(column+1) - row;

处于首尾行的元素，其索引值满足以下规律：

去掉空列，可以得到如下的一张表：

行\列 第0列 第1列 第2列
第0行 0 8 16
第4行 4 12 20

通过分析此表，可以得出，其索引值可以用公式计算得到：
2*(N-1)*column + row

特殊情况处理

  • 当N<=0时，输入无效，此时不做转换，输出源字符串即可
  • 当N=1时，仅仅转换成一行字符串，可以输出源字符串即可
  • 当字符串长度strlen(s)< N时，转换后，仅仅排成一列，也可以不做转换，输出源字符串即可。
